# Plan for Refactoring `src/background.js` into Services

## Goal
Break down the large `src/background.js` file into smaller, more manageable services to improve maintainability and readability.

## Proposed Architecture

### 1. `services/listeners.service.js`
- **Purpose**: Handle all event listeners related to proxy changes and browser interactions.
- **Contents**:
    - `ListenersService` class.
    - Logic for handling message passing, tab reloading, etc.
    - Exported instance: `export const listenersService = new ListenersService(...)`.

### 2. `services/storage.service.js`
- **Purpose**: Encapsulate all storage operations (`local`, `sync`).
- **Contents**:
    - `StorageService` class.
    - Methods for getting/setting state, logs, and sync data.
    - Exported instance: `export const storageService = new StorageService(...)`.

### 3. `services/icons.service.js`
- **Purpose**: Handle icon generation and updates.
- **Contents**:
    - `IconsService` class.
    - Logic for creating dynamic icons, updating action icons, etc.
    - Exported instance: `export const iconsService = new IconsService(...)`.

### 4. `services/notifications.service.js`
- **Purpose**: Encapsulate notification logic.
- **Contents**:
    - `NotificationsService` class.
    - Methods for showing failover notifications, permission notifications, etc.
    - Exported instance: `export const notificationsService = new NotificationsService(...)`.

## Refactoring Steps

1.  **Analyze current dependencies**: Identify what each function needs (e.g., `api`, `state`, `addLog`).
2.  **Create `services/` directory**.
3.  **Implement `ListenersService` service**:
    - Move relevant logic from `background.js`.
    - Ensure it receives necessary dependencies via constructor or methods.
4.  **Update `src/background.js`**:
    - Import the new services.
    - Replace inline function calls with service method calls.
5.  **Verify functionality**: Ensure all event listeners and state updates still work as expected.
